## Summary

**v2.1.2 -- two follow-on bug fixes** from real-use feedback minutes
after v2.1.1 shipped.

### /code bar colors

The `/code` project cards have a small "type breakdown" bar chart
inside each card. After the v2.1.1 palette refactor, those bars
went blank (4px transparent strips).

Root cause: a duplicate `.bar-fill` CSS rule in the /code section
of app.css predates the palette refactor. It sits later in the
file than the palette-driven generic `.bar-fill` rule and
overrides the `background` declaration with nothing. The
dashboard (which uses the same class) was fine because no second
rule shadowed it.

Fix: scoped the duplicate rule under `.code-project-card-bars`
and added an explicit `background: var(--type-color, ...)`. Both
surfaces now paint consistently from the same single-source
palette.

### Nebula deselect lag

Click empty canvas (or press Escape) to deselect a node, and the
graph stuttered visibly for ~1s before settling. Two compounding
causes:

1. **Transition fan-out.** The base `node` selector had
   `transition-property: 'opacity, border-width, underlay-opacity,
   underlay-padding'` with a 220ms duration. The `.dim` class
   toggles both `opacity` and `underlay-opacity`, so deselecting
   on a 478-node graph fired **471 simultaneous opacity tweens**.
   With motion-blur on, every redrawn frame for 220ms touched all
   471 nodes.

2. **Pulse animation kept running.** `_stopPulse()` only unset a
   guard flag. The in-flight `node.animate({underlay-padding,
   underlay-opacity}, 900ms)` chain on the previously-selected
   node kept tweening for up to 900ms after deselect, with its
   `complete` callback queueing the next half of the cycle BEFORE
   the flag check fired. So the node kept mutating styles long
   after it should have stopped.

Fixes:

- Dropped `opacity` and `underlay-opacity` from the base
  `transition-property`. They're the properties `.dim` toggles
  across the whole graph; snapping them is fine. The selected
  cluster still feels "lifted" because `border-width` and
  `underlay-padding` (changed only on the 1-or-few `.hl` nodes)
  still transition.
- `_stopPulse()` now calls `node.stop(true, false)` to cancel
  the queued animations AND `removeStyle('underlay-padding
  underlay-opacity')` to clear the inline styles the pulse wrote.
- `deselect()` calls `_stopPulse()` FIRST, then
  `cy.elements().stop(true, false)` to cancel any other queued
  animations (e.g. selectFromCanvas's camera-fit), THEN runs the
  bulk `removeClass('hl dim')` inside `cy.batch()`.

Net effect on a 478-node graph: deselect sync ~10ms (was ~90ms);
no lingering tween animations after 300ms (was running until the
900ms pulse complete callback fired).
