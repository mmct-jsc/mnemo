## 0. Why this release exists

mnemo *informs* an agent (ranked, cited knowledge every prompt) but cannot
*govern* one. There is no way to declare "you MUST use a conventional commit
prefix", "you MUST NOT touch `main` directly", "code review is required
before commit", or "this verification must pass" -- and have mnemo surface it
at the right moment and *enforce* it. The conventions that exist (CLAUDE.md
prefixes, no-emoji, the implement-platform phases, mandatory RCA) are PROSE,
unenforced; an agent skips them silently.

v6.1.0 makes governance first-class: coding constraints, workflow gates,
specs, mandatory verification, and code review become citable,
machine-readable **rule** nodes that mnemo (1) surfaces *prescriptively* and
*context-aware*, and (2) **enforces with teeth** -- blocking a tool call or
session end until a mandatory step is *provably* satisfied.

## 1. Owner-locked decisions (2026-06-18)

| # | decision |
|---|----------|
| Enforcement | **Hard enforcement in v1** -- wire the unused PreToolUse (deny/ask) + Stop (block) hooks. Default mode `warn`; `block` opt-in per rule. |
| Authoring | **Hand-written rule files** -- one `.md` per rule with structured frontmatter. No CLAUDE.md auto-import / codebase-generated rules in v1. |
| Gate honesty | **Evidence-based** -- a mandatory step counts as done only when mnemo captures real proof (the verify command's actual exit code, the review actually run), never the agent's assertion. Aligns with the operator-green completion rule. |

## 2. Design principle (non-negotiable)

Reuse mnemo's primitives, don't fork them. Rules are nodes (the
`frontmatter_json` seam + `base` flag); context surfacing is a retrieval
finisher (like `exact_name_boost`); evidence is a table modeled on
`audit_queue`; code checks (G5) are a `LENS_DETECTORS` entry feeding the
existing queue + confirm-then-apply. **Every hook stays fail-open**: daemon
down / store error / ledger error -> exit 0, never block. A governance layer
that bricks a session when mnemo is unavailable is worse than none.

## 3. Rule schema (frontmatter, carried in `Node.frontmatter_json`)

`type: rule` parses with ZERO parser change: `ingest._resolve_type`
(ingest.py:175) returns any bare `type:` already in `NODE_TYPES`.

```yaml
---
name: conventional-commit-prefix
type: rule
base: true                       # reuse existing flag = applies to every project
description: Commit messages must use a conventional prefix.
rule:
  id: rule.commit.prefix         # stable human id (citation-friendly)
  modality: MUST                 # MUST | MUST_NOT | SHOULD  -> retrieval priority + inject bypass
  enforcement: block             # inform | warn | require-ack | block (orthogonal to modality)
  applies_to:
    glob: ["**/*"]               # file-path triggers
    intent: ["feedback-recall"]  # task-type triggers (reuse intent.py tags)
    tool: ["Bash"]               # tool-call triggers (PreToolUse)
    tool_arg_match: "git commit" # regex/substring on tool input
  verify: { command: "uv run ruff check .", expect_exit: 0 }
  requires_step: review          # gate rules: review | verify | ack (checked vs the ledger)
  links: { refines: rule.commit.style }
---
Body: the rule text, rationale, and escape hatch.
```

`modality` drives retrieval priority + injection bypass; `enforcement` drives
the hook spine (orthogonal -- a MUST can be inform-only during rollout);
`applies_to` is the context key; `verify`/`requires_step` power evidence
gates.

## 4. Phases (each red->green->commit; G1-G4 = v6.1.0)

### G1 -- rule representation + authoring
- `store.py`: add `"rule"` to `NODE_TYPES`; add `"refines"` to `EDGE_RELATIONS`.
- `ingest.py`: `_resolve_rule_meta(fm)` validates/normalizes the `rule:` block into `frontmatter_json` (unknown modality/enforcement -> downgrade to `inform` + warn, never raise). Wire `links.*` into the edge linker.
- `governance.py` (NEW): `Rule` dataclass, `parse_rule(node)`, `rule_applies(rule, *, glob_path, intent_tags, tool_name, tool_arg)`, `modality_rank`. Pure, no I/O.
- `skills/mnemo-rule-add/SKILL.md`: scaffold a rule file.
- DoD: a rule file ingests as a `rule` node; `rule_applies` truth table green; malformed rule fails open.

### G2 -- context-aware prescriptive injection (implements v6 cut A)
- `api_schemas.py`: optional `file_paths`/`tool_name`/`tool_arg` on `QueryIn`; `rules` section on `QueryOut` (defaults None -> byte-stable).
- `retrieve.py`: governance finisher -- multiply applicable `rule` candidates by `cfg.governance_boost` (~2.0x), beside the finishers at retrieve.py:252,257.
- `compress.py`: partition into mandatory (applicable MUST/MUST_NOT) vs ranked; mandatory get a reserved token slice, emitted first, never dropped by budget.
- `cli.py` (`hook_user_prompt_submit` formatter at 1059-1065): two sections -- `## Active rules (mnemo) -- binding` (capped <=5) then `## Relevant memory (mnemo)` as the lean pointer (this is the v6 injection-as-pointer cut).
- DoD: applicable MUST_NOT injected even at budget=20; non-applying excluded; pointer under cap; boost ordering.

### G3 -- evidence ledger + capture (makes "mandatory" real)
- `store.py`: `governance_ledger` table (modeled on `audit_queue`): `(session_id, rule_id, step, status, evidence, touched_files, stamped_at)` + `record_evidence`/`gate_satisfied`/`record_touched_file`.
- `cli.py` (`hook_post_tool_use` at 1068): broaden to capture `Bash` -- read `tool_input.command` + `tool_response` exit; if it matches an active rule's `verify.command` and exit == `expect_exit`, record an evidence row (real exit code). Record every Edit|Write `file_path` as a touched file. **The agent cannot fake it -- mnemo reads the actual tool result.**
- `hooks.json`: PostToolUse matcher `Edit|Write` -> `Edit|Write|Bash`.
- `agent_tools.py`: `mnemo_check` (risk=safe, runs a verify command + records evidence) + `mnemo_rules` (risk=safe, list applicable rules). Regen wire snapshot 30 -> 32.
- DoD: matching exit 0 records satisfied; non-zero doesn't; `gate_satisfied` needs a covering row; touched-file tracking.

### G4 -- hard enforcement: PreToolUse deny + Stop block
- `governance.py`: `evaluate_gate(tool_name, tool_arg, session_id, store) -> Decision`.
- `cli.py`: `hook pre-tool-use` (emit `hookSpecificOutput.permissionDecision = deny|ask` + reason; if `governance_enforce_mode != block` -> warn via additionalContext + allow) and `hook stop` (block/ask on unsatisfied mandatory step for touched files).
- `hooks.json`: wire `PreToolUse` + `Stop`.
- `config.py`: `governance_enforce_mode` (`off|warn|block`, default `warn`); honor `MNEMO_GOVERNANCE_BYPASS=1` (mirrors `MNEMO_NO_SESSION_BANNER`).
- Fail-open: any error -> exit 0, no decision.
- DoD: deny only when an applicable block-rule's step is unsatisfied; bypass flips deny->allow; daemon-down -> fail-open; Stop scoping; satisfied row clears the block.

## 6. Definition of Done (v6.1.0)
1. `governance.py` pure core + `rule` node type + `refines` edge, unit-tested; malformed rules fail open.
2. Prescriptive injection: applicable binding rules surface in the `## Active rules` block; general knowledge demoted to the lean pointer (v6 cut A done).
3. Evidence ledger: mandatory verify is satisfied only by a captured real exit code; touched files tracked.
4. Hard enforcement: PreToolUse denies / Stop blocks on an unsatisfied applicable `block` rule; default `warn`; bypass works; fail-open verified live.
5. Full suite green; ruff clean; MCP wire snapshot updated (32 tools) + byte-stable for the existing 30.
6. Shipped operator-green dual-remote; daemon healthy on 6.1.0; live dogfood (deny -> verify -> allow -> bypass -> daemon-down fail-open) passes; handover written.

## 7. Risks
- False-positive blocks erode trust -> default `warn`, `block` opt-in, loud reason naming the bypass, conservative matchers.
- Hook latency on every tool call -> daemon-first warm path; `rule_applies` is regex/glob; fail-open if slow.
- Gate theater -> resolved by evidence-based capture (real exit codes, not assertions).
- Over-injection -> mandatory inline cap <=5; SHOULDs pointed-to not injected.
