## 3. Architecture

### 3.1 Page-context provider contract
Every interactive page defines `window.mnemoPageContext()` → a small
JSON object. base.html ships a default
(`{ page, path }` from the nav `page`). Pages override:
- **graph.html (Nebula):** `+ { selected_node_id, visible_node_ids
  (cap ~60), query, workspace }` from the live Cytoscape/cosmos state.
- **settings:** `+ { weights, k, recent_feedback: {pos,neg,n} }`.
- **/code, /nodes:** `+ { selected_node_id }`.
`mnemoChat()` (shared) calls `window.mnemoPageContext()` at send time
and PATCHes it onto the conversation (`page_context`) before each run,
so the model always sees the current screen. (Server: `page_context`
already persists on `chat_conversations`; just keep it fresh.)

### 3.2 `mnemo_page_context` (safe read tool)
Returns the server view + the client-supplied context (passed through
the message envelope). Lets the agent ground "what's related in this
session / on this page" without guessing.

### 3.3 In-page action listeners (the real fix for "not just a link")
Add, per page, listeners for the existing CustomEvents:
- graph.html: `mnemo-select-node` → focus+highlight+center that node
  on the live graph; `mnemo-set-filter` → drive the type/confidence/
  layout controls; `mnemo-open-panel` → open the detail panel.
- settings: `mnemo-open-panel`/a new `mnemo-retune` → focus + (confirm)
  apply weight deltas on the retune panel.
- **`[mnemo:id]` cite-links become context-aware** (chat.js
  `_citeLinks` + a delegated click handler): on a page that can show
  the node in place (Nebula/code) dispatch an in-page action; else an
  inline preview popover (reuse `previewMarkup`); full-page
  `/node/<id>` is only the final fallback. NO blind redirect.

### 3.4 Nebula "related in this session"
New safe tool `mnemo_session_nodes(conversation_id)` → the cited/used
node ids of the conversation + their 1-hop neighbours (reuse
`store.get_edges_for_nodes`). The agent calls it then emits
`mnemo-set-filter`/`mnemo-select-node` to highlight them live.

### 3.5 Settings retune assistant
`page_context` carries weights/k + feedback stats. Agent reads memory
(`mnemo_query`) + proposes retune deltas, explains tradeoffs, and
(confirm-risk `mnemo_apply_retune`) applies via the existing retune
endpoint; the page validates by showing before/after.

## 5. Bubble fix — root-cause notes (so P1 is fast)

Reverted WIP put `.mnem-bubble { position:fixed; width:max-content;
max-width:min(260px, calc(100vw-24px)) }` + `_anchorToLauncher`.
Screenshot showed the bubble clipped to ~"Want m to". Likely cause:
`_anchorToLauncher` read `el.offsetWidth` right after `x-show` flipped
display (no layout yet → tiny/wrong width) AND `width:max-content`
gave the box no stable width, so the clamp math jammed it against the
edge and the text hard-wrapped. P1 fix = definite width + anchor only
in `_repositionSoon`'s rAF/timeout (post-layout) + `offsetWidth||280`
fallback. It's small; the value is doing it with the same live-geometry
verification discipline as the panel anchoring (gotcha #25), not by
eye.
