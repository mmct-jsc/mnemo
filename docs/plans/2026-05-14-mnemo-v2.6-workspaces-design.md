# mnemo v2.6 design — Workspaces + indexing safeguards

**Codename:** Workspaces. **Predecessor:** v2.5.1 (v2-deferred sweep closed).
**Blocks:** v3.0 chat (v3's retrieval scope reads from the active workspace).
**Status:** design, brainstormed + validated 2026-05-14.
**Owner:** mnemo core.

## 1. Goals + non-goals

mnemo gains **named workspaces** that bundle a project-key set + filter prefs
into a switchable unit. Workspaces become the durable answer to "what's in
scope for this session?" — replacing the implicit single-active-project
pointer with an explicit, multi-project, user-named selection. Indexing
safeguards layer on: gitignore-respecting walks, hard size caps, malformed +
suspicious file detection with user-in-the-loop decisions that persist.

### Hard goals (v2.6.0 must ship)

1. **`workspaces` SQLite table** + active-workspace pointer; full CRUD via
   `/v1/workspaces`.
2. Top-bar **workspace switcher** dropdown with recent-list + "+ New
   workspace" + the active workspace's project chips.
3. **BASE-only UI mode** when no workspace is active. Every page (Nebula /
   /code / Search / future /chat) filters to BASE-flagged nodes. Sources +
   Settings remain workspace-independent.
4. **Soft + hard workspace caps** (75k / 200k nodes; refuse switch above
   hard).
5. **Auto-router dual-source proposal** on add: dry-run says "I found docs +
   code; register as (a) docs_dir / (b) code_repo / (c) BOTH"; user picks.
6. **`.gitignore` auto-merge** into source `exclude` patterns at
   registration.
7. **Reindex report** with three categorized sections (auto-skipped /
   malformed / suspicious) + per-file user decisions persisted in a new
   `source_overrides` table.
8. **Suspicious-file detector** with 7 heuristics (binary masquerade,
   suspected secrets, generated-header marker, lock/minified/snapshot,
   high-entropy, repeated-line bloat, build-output names).
9. **Incremental reindex on workspace activation** — no full re-walk; only
   changed-since-last files via existing hash-diff watcher logic.

### Theme + coding-pattern requirements (enforceable)

- All CSS reuses existing variables (`--accent`, `--panel-2`, `--fg`,
  `--fg-muted`, `--border`, `--radius`, `--mono`) — no new color literals.
- Alpine.js: named factory pattern (`x-data="workspaceSwitcher()"`); no
  double-init (`x-data` + `x-init="init()"` is documented anti-pattern in
  `feedback_alpine_double_init.md`).
- Single-source palette (pipeline 19): workspace project chips reuse
  `.nebula-chip` + `.swatch-*` from `palette.py`.
- Backdrop-blur + glassmorphic feel matches `.nebula-detail`.
- Animations via CSS `@keyframes` (pipeline 18 — DOM overlay over canvas-
  library); `prefers-reduced-motion` short-circuits per v2.2 contract.
- Stagger reveal on dropdown "recent" list uses `mnemoStaggeredReveal`
  (v2.2.0 primitive).
- Reuse `.dash-card` / `.dash-card-head` patterns for `/workspaces`
  management page tiles.
- Typography hierarchy preserved (h3 1.15rem, h4 0.85rem, etc.).
- All templates extend `base.html`; CSS lives in `app.css` (no per-page CSS
  files).
- New code: ruff-clean, type hints, pytest surface tests over JS execution
  (matches v2.2 test pattern).

### Hard non-goals (deferred to later minors)

- Workspace export / import as portable JSON (v2.6.x).
- Per-workspace embedding-quality tuning (v2.7+).
- Workspace sharing / federation (v3+ team feature).
- Auto-promoting frequently-accessed nodes to BASE (v3.x heuristic).
- No cloud sync of workspaces.
- No auth / multi-user.
- No automatic workspace creation (always user-initiated).

## 2. Architecture + schema

### Three new SQLite tables (additive via `_ensure_columns`)

```sql
CREATE TABLE workspaces (
    id                TEXT PRIMARY KEY,         -- uuid4 hex
    name              TEXT NOT NULL UNIQUE,     -- user-supplied; UI prevents collision
    project_keys      TEXT NOT NULL,            -- JSON array
    filter_prefs      TEXT,                     -- JSON: { layout?, typeFilters?, minConfidence?, ... }
    page_state        TEXT,                     -- JSON: { last_page?, last_node_id?, ... } -- v3 chat reads this
    created_at        INTEGER NOT NULL,
    updated_at        INTEGER NOT NULL,
    last_activated_at INTEGER                   -- drives "recent" ordering in the switcher
);

CREATE TABLE workspace_state (
    -- Single-row table holding the active workspace pointer.
    -- Empty / NULL means "no workspace active = BASE-only mode".
    singleton    INTEGER PRIMARY KEY CHECK (singleton = 0),
    active_id    TEXT REFERENCES workspaces(id) ON DELETE SET NULL,
    activated_at INTEGER
);

CREATE TABLE source_overrides (
    -- Per-path decisions from the reindex report's malformed +
    -- suspicious sections. (source_path, decision) is the durable
    -- contract; reindex consults this table before classifying.
    source_path TEXT PRIMARY KEY,
    decision    TEXT NOT NULL,           -- 'always_skip' | 'always_keep' | 'retry'
    reason      TEXT,                    -- 'malformed:bad_yaml' | 'suspicious:secret' | ...
    decided_at  INTEGER NOT NULL
);
```

### Reindex reports — transient

Stored in `app.state.last_reindex_report` (Python in-memory) until the next
reindex run replaces it. UI fetches via `GET /v1/reindex/report`. Cheap; no
schema cost.

### Module layout (new under `daemon/mnemo/`)

- `workspaces.py` — CRUD + activation logic + safety caps. Reads
  `chat_permissions` (from v3) for cross-feature respect.
- `safeguards.py` — pure functions: `classify_file(path, content_bytes)`
  returning `'ok' | 'auto_skip:<reason>' | 'malformed:<reason>' |
  'suspicious:<reason>'`.
- `ingest.py` — `reindex_events` consults `source_overrides` before
  classifying; emits new `'classified'` event per file + a `'report'` event
  before `'done'`.
- `auto_router.py` — `propose_source(path)` returns `[docs_dir_proposal,
  code_repo_proposal]` for the dual-source add UX.
- `server.py` — `/v1/workspaces/*`, `/v1/source_overrides/*`,
  `/v1/reindex/report`, `/v1/sources/propose`, `/v1/events` endpoints.

### Active-workspace broadcast

`POST /v1/workspaces/<id>/activate` (a) updates `workspace_state.active_id`,
(b) emits an SSE event on the new `/v1/events` channel that any open browser
tab subscribes to, (c) kicks off an incremental reindex of the workspace's
project_keys' sources. UI listens via `mnemoStreamFromSSE` + re-fetches data
scoped to the new active set.

## 3. Add-source UX (auto-router dual proposal + .gitignore)

### Two-step flow

```
1. POST /v1/sources/propose { path }
     -> 200 { proposals: [
          {kind: 'docs_dir',  include_count: 247,  est_nodes: 247,   sample: [...]},
          {kind: 'code_repo', include_count: 1820, est_nodes: 4350,  sample: [...]},
        ],
        gitignore_excludes: ['node_modules/**', 'dist/**', ...],
        gitignore_files_found: ['.gitignore', 'src/.gitignore'],
        warnings: [
          {kind: 'large_project', message: '6,200 indexable files -> may hit cap'},
        ],
      }

2. POST /v1/sources { path, kind, include?, exclude?, project_key? }
   (repeat per accepted proposal -- user can pick one, both, or neither)
```

### Auto-router heuristic for the proposal

- Walks the path with bundled tree-sitter extensions + markdown / PDF / txt.
- Qualifies as `code_repo` if ≥10 source files match tree-sitter extensions.
- Qualifies as `docs_dir` if ≥3 markdown / PDF / txt files match.
- BOTH proposals fire when both qualify (common case: project repo with
  `docs/` folder). User then registers two sources, each scoped via
  auto-generated `include` patterns (`docs_dir` gets `include="docs/**/*.md,
  README.md"`, `code_repo` gets `include="src/**,!docs/**"`).

### `.gitignore` handling

- `propose` walks the path for every `.gitignore` (incl. nested via
  pathspec's gitignore syntax) and returns the merged exclude list.
- On `POST /v1/sources`, the daemon **prepends** the gitignore-derived
  patterns to the user-supplied `exclude` field. Stored verbatim in the
  source row.
- A "Refresh .gitignore" button on the source edit page re-reads + merges
  (preserves user-added custom excludes).

### Notification

The UI dry-run preview IS the notification. The user sees "I found X docs +
Y code files" before clicking add — there's no path where the user adds
without seeing what's getting indexed. The original gap ("any notify that
this only index doc, not whole") is closed structurally.

## 4. Reindex report + skip semantics

### New event types in `reindex_events`

```python
('classified', {
    idx: int,
    path: str,
    category: 'indexed' | 'auto_skipped' | 'malformed' | 'suspicious',
    reason: str,                  # e.g. 'oversize:7.2MB', 'malformed:bad_yaml',
                                  #      'suspicious:suspected_secret'
    override_applied: bool,
})
('report', {                      # one final emit BEFORE 'done'
    auto_skipped: [{path, reason}, ...],
    malformed:    [{path, reason, retry_hint}, ...],
    suspicious:   [{path, reason, heuristic}, ...],
    indexed_count: int,
    duration_ms: int,
})
```

### Classification pipeline (`daemon/mnemo/safeguards.py`)

```python
def classify_file(path: Path, source: Source, content_bytes: bytes | None) -> Classification:
    # Order matters: cheapest checks first.
    if override := load_override(path):
        return Classification(category=override.decision_to_category(), reason=...)
    if matches_gitignore(path, source) or matches_default_skip(path):
        return AUTO_SKIPPED('gitignore')
    if path.stat().st_size > 5 * 1024 * 1024:
        return AUTO_SKIPPED(f'oversize:{path.stat().st_size}')
    if path.suffix not in SUPPORTED_EXTS:
        return AUTO_SKIPPED('unsupported_ext')
    if content_bytes is None:
        content_bytes = path.read_bytes()
    # Suspicious heuristics (each a pure function; first match wins)
    for check in (
        is_binary_masquerading_as_text,    # > 30% non-printable / NUL bytes
        looks_like_secret,                 # API key + private key + bearer + .env shapes
        has_autogenerated_header,          # // @generated, # AUTOGENERATED, etc.
        is_lock_minified_or_snapshot,      # *.lock, *.min.{js,css}, *.snap, *.bundle.*
        is_high_entropy_blob,              # Shannon entropy > 7.5 over text-typed file
        has_repeated_line_bloat,           # > 80% lines repeat, file > 100KB
        is_build_output_name,              # bundle.js / vendor.js / dist/* / build/*
    ):
        if reason := check(path, content_bytes):
            return SUSPICIOUS(reason)
    # Try the actual parse:
    try:
        parsed = parse(path, source, content_bytes)
        return INDEXED(parsed)
    except ParserError as exc:
        return MALFORMED(f'{exc.kind}:{exc.msg}')
```

### Per-file decisions are durable

When the user picks `always_skip` / `always_keep` / `retry` in the report
UI, the choice writes to `source_overrides` keyed by source_path. Next
reindex consults the table first and skips re-classification for known
paths.

### `POST /v1/source_overrides` (batch)

Body: `[{source_path, decision}, ...]`. Triggers a partial reindex on the
affected paths so the user's decisions take effect immediately.

## 5. Workspace UX

### Top-bar workspace switcher (NEW shared header via `base.html`)

```
+------------------------------------------------------------+
| mnemo  [Workspace: ai-edge work  ▾]  ...  [Mnem avatar]   |
+------------------------------------------------------------+
                          │
                          ▼ click → dropdown:
                  ┌─────────────────────────┐
                  │ Active                  │
                  │ ● ai-edge work          │
                  │   3 projects · 12k nodes│
                  │                         │
                  │ Recent                  │
                  │ ○ mnemo dev   (2h ago)  │
                  │ ○ soi work    (yest.)   │
                  │ ○ knowledge-base (3d)   │
                  │                         │
                  │ ─────────────────────── │
                  │ + New workspace         │
                  │ Manage workspaces...    │
                  │ ─────────────────────── │
                  │ ○ No workspace          │
                  │   (BASE-only mode)      │
                  └─────────────────────────┘
```

Clicking a workspace fires `POST /v1/workspaces/<id>/activate`. The daemon
updates state, kicks off incremental reindex, and broadcasts an SSE event on
`/v1/events`. Every open browser tab subscribes via `mnemoStreamFromSSE` and
re-fetches its data. The current-page Alpine component re-runs its load.

### "+ New workspace" modal

Multi-select project chips (list of known `project_key`s across all
sources) + name field + optional starting filter prefs (clone-from-existing
dropdown). Submit → `POST /v1/workspaces` + auto-activates the new
workspace.

### "Manage workspaces..." page (`/workspaces`)

Full CRUD. Per-workspace card shows name, project chips, node count (live
computed), `last_activated_at`, filter prefs preview, Edit / Duplicate /
Delete actions. Delete prompts twice if active; activates "No workspace"
after delete.

### No-workspace UI mode

- Top bar: `Workspace: — (BASE-only)` with a "Pick a workspace" CTA.
- Nebula: only BASE-flagged nodes (~10-20 typical).
- /code: empty state with "Pick a workspace" button.
- Search: BASE hits only.
- /chat (v3): launches but agent sees BASE-only context.
- Sources + Settings + /workspaces: fully functional (workspace-
  independent).

### Hard-cap refusal UX

`POST /v1/workspaces/<id>/activate` returns `409 WorkspaceTooLarge` with
payload `{total_nodes, hard_cap, projects: [{key, nodes}, ...]}`. The
switcher renders a modal: "Workspace 'big-monorepo' has 247,000 nodes — over
the 200k cap. Remove a project to activate." with checkboxes next to each
project. User unchecks + retries.

### Soft-cap warning UX

Activation succeeds; the switcher shows a yellow chip "Large workspace ·
retrieval may be slower" next to the workspace name in the top bar.
Clickable → info panel + "Don't show again" toggle (per-workspace).

## 6. API surface

### New endpoints

```
# Workspaces
GET    /v1/workspaces                     List (active first, then by last_activated_at DESC)
POST   /v1/workspaces                     Create. Body: {name, project_keys, filter_prefs?, page_state?}
GET    /v1/workspaces/<id>                Read one (includes live node_count + warnings)
PATCH  /v1/workspaces/<id>                Update name / project_keys / filter_prefs
DELETE /v1/workspaces/<id>                Delete (if active, deactivates first)
POST   /v1/workspaces/<id>/activate       Activate. 409 if hard cap exceeded
POST   /v1/workspaces/clear               No-workspace mode (clear active pointer)
GET    /v1/workspaces/active              Read active pointer + payload

# Source proposals (auto-router dual proposal)
POST   /v1/sources/propose                Body: {path}. Returns proposals + gitignore_excludes + warnings
POST   /v1/sources                        Existing endpoint extended: accepts gitignore-merged excludes

# Skip decisions
GET    /v1/source_overrides               List overrides (sortable by decided_at)
POST   /v1/source_overrides               Batch upsert. Triggers partial reindex of affected paths
DELETE /v1/source_overrides/<source_path> Remove override

# Reindex report (in-memory, transient)
GET    /v1/reindex/report                 Most recent report; 404 if none yet this daemon run

# Broadcast (new SSE channel)
GET    /v1/events                         SSE: 'workspace_activated', 'workspace_deleted',
                                          'reindex_started', 'reindex_done'.
```

### Integration with v2.2 streaming surface

`reindex_events` keeps its event protocol but adds `('classified', ...)` per
file + `('report', ...)` before `('done', ...)`. The Sources page progress
UI from v2.2.0 phase 3 grows a "View report" button that appears on `done`,
opening the three-section report modal.

### Workspace activation flow over the wire

```
1. UI:     POST /v1/workspaces/abc123/activate
2. Daemon: validate caps -> if hard, return 409 WorkspaceTooLarge
3. Daemon: update workspace_state.active_id; emit 'workspace_activated' on /v1/events
4. Daemon: kick off incremental reindex for workspace's projects (returns immediately; reindex runs in bg)
5. UI:     receives SSE event; re-fetches scoped data; banner "Reindexing 'ai-edge work'..."
6. Daemon: emits 'reindex_done' when complete; UI clears banner
```

```json

    "hard_cap_nodes": 200000,

  }
}
```

## 7. Testing + phased roadmap

### Test strategy (matches v2.x convention)

| Layer | Test approach |
|---|---|
| `workspaces.py` (CRUD, activation, cap-checking) | Unit tests with tmp store; assert create / list / get / patch / delete; activate triggers `workspace_state.active_id` update; soft cap warns but allows; hard cap raises `WorkspaceTooLarge`; deleting active clears pointer |
| `safeguards.py` (classify_file + 7 heuristics) | Pure-function unit tests per heuristic. 2-3 cases each (positive, negative, edge). <5ms each |
| `auto_router.propose_source` | tmp_path fixture with mixed docs + code; assert: docs-only repo emits one proposal; code-only emits one; mixed emits BOTH; gitignore parsed |
| `ingest.reindex_events` (classified + report events) | TestClient over POST /v1/reindex/events SSE; assert events appear in order; report has three sections populated correctly |
| `source_overrides` enforcement | Set an `always_skip` override; reindex; assert file is skipped without re-classification |
| Endpoints | TestClient over the FastAPI app, same shape as v2.x endpoint tests |
| UI (top-bar switcher + /workspaces + report modal) | Surface tests grep templates for Alpine state, factory names, x-effects (pattern from `test_progressive.py`) |
| Live UX | Manual via preview tool (pipeline 8) — workspace switch flow, hard-cap modal, three-section report |

### Open questions deferred to v2.6.x / v2.7

- Workspace **export / import** as portable JSON (cross-machine workflows).
- **Per-workspace embedding-quality tuning** (different retune weights per
  workspace).
- Workspace **sharing / federation** (v3+ team feature).
- **Auto-promote to BASE** heuristic — frequently-accessed nodes across
  workspaces become BASE candidates.
- **Workspace presets** (clone-from-existing with stronger templating).
- **Per-source malformed retry policy** — `retry` decisions could
  back-off (try, then weekly, then monthly).
- **Sensitive-content redaction** — when a suspected-secret file is kept,
  redact the matched regions before embedding (needs embedder to accept
  redacted bytes).

### Hard non-goals (re-stated)

- No cloud sync of workspaces.
- No auth / multi-user.
- No workspace-scoped settings overriding global settings (only filter prefs
  are workspace-scoped; provider keys + companion settings stay global).
- No automatic workspace creation from heuristic (always user-initiated).
