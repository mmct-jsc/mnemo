# mnemo v5.25.0 -- Discover & Presence (make-it-real arc, workstream B)

> DoD-first design doc (2026-06-09). Per-release spec; rides with the
> v5.25.0 release branch (committed, per pipeline #21). The umbrella arc
> roadmap (`2026-06-09-mnemo-real-install-discover-trust-design.md`) stays
> LOCAL-only. Workstream A (install & register) shipped as v5.24.0.

## 1. Goal

Workstream A made mnemo a registered Claude Code plugin (commands appear,
hooks fire). But the user's complaint had two halves; A fixed "no commands
show up", B fixes **"no notify, nothing related to mnemo -- it lives
alongside other tools invisibly."** v5.25.0 makes mnemo *visibly present*
to the human in Claude Code, and *discoverable* to the model in the other
six MCP-only hosts.

Scope confirmed with the user: **all of workstream B in one release.**

## 2. Verified Claude Code contract (locked before building)

Confirmed via the claude-code-guide agent against current CC docs
(`hooks.md`, `statusline.md`), high confidence:

- **Hook JSON output (SessionStart).** A command hook may print a JSON
  object to stdout (exit 0 required). Relevant fields:
  - `systemMessage` (top-level, string) -- **shown to the USER** as a
    notice; NOT injected into the model. This is the "notify" channel.
  - `hookSpecificOutput.hookEventName` (must equal `"SessionStart"`) +
    `hookSpecificOutput.additionalContext` (string) -- injected into the
    model (the current behaviour). `systemMessage` and `additionalContext`
    are independent and may coexist in one object.
  - Caveat: malformed JSON has NO documented fallback. So the hook MUST
    always `json.dumps` a dict (never hand-build) and FAIL OPEN (emit
    nothing, exit 0) on any exception.
- **statusLine is NOT plugin-ownable.** It lives only in user/project
  `settings.json` under the `statusLine` key (`{type:"command",
  command, padding?, refreshInterval?}`). A plugin cannot declare one; the
  installer must write it into the user's settings. The command receives a
  JSON payload on stdin with `session_id`, `cwd` / `workspace.current_dir`,
  `workspace.project_dir`, `model.display_name`. Output = stdout (one line
  used; ANSI colour supported). Refresh debounced ~300ms; `refreshInterval`
  min 1s.

## 3. Components

### Component 1 -- user-visible session-start banner (the "notify" fix)

**File:** `daemon/mnemo/cli.py` `hook_session_start()` (~L727).

Change the output from raw `typer.echo(text)` to a JSON object:

```python
payload = {
    "systemMessage": f"mnemo: {total:,} memories across {sources} source(s) "
                     f"-- /mnemo-query to recall",
    "hookSpecificOutput": {
        "hookEventName": "SessionStart",
        "additionalContext": <the existing memory-map text>,
    },
}
typer.echo(json.dumps(payload))
```

- `systemMessage` is one line, no emoji, <= ~90 chars.
- The existing memory-map text is preserved verbatim as `additionalContext`
  (zero change to what the model sees).
- Opt-out: gate the `systemMessage` on a config flag
  `config.get("session_banner", True)`. When false, emit only
  `hookSpecificOutput` (model context, no user notice).
- Fail-open unchanged: any exception -> `return` (exit 0, no output).

**Tests** (`test_hook_cli.py`, update existing): parse the echoed stdout as
JSON; assert `systemMessage` contains "mnemo" and "/mnemo-query", and
`hookSpecificOutput.additionalContext` still contains "memory map" +
version. Add: banner suppressed when `session_banner=False`. Add: a
forced-exception path emits nothing + exit 0.

### Component 2 -- statusline (persistent presence)

**New CLI:** `mnemo statusline` (`cli.py`). Reads the CC payload from stdin
(defensive parse; tolerate empty/garbage), prints ONE line:

- Probe the daemon `/v1/health` with a ~250ms timeout (do NOT open the
  SQLite store in this hot path -- avoid lock contention + keep it instant).
  - reachable -> `mnemo {compact_count}` (e.g. `mnemo 17.9k`), optionally
    `+ up{N}` when a per-session inject-count file exists (see below).
  - unreachable -> `mnemo offline`.
- Compact-count helper: `17919 -> "17.9k"`, `<1000 -> str(n)`.
- Pure formatting split out as `format_statusline(health|None, inject|None)`
  so it is unit-testable without a live daemon.

**Inject-count signal (graceful enhancement).** The user explicitly wanted
"injected N memories". The `user-prompt-submit` hook already computes
`len(result.hits)`; have it also write `{ "count": k }` to a per-session
file keyed by `session_id` under the mnemo state dir (read `session_id`
from the hook's stdin JSON). `mnemo statusline` reads that file by
`session_id` and appends `up{N}`. Best-effort: missing/garbage file -> just
omit. Opportunistic cleanup: prune session files older than 24h on write.

**Installer:** `install.sh` / `install.ps1` -- after MCP registration, add a
`statusLine` entry to the user's `settings.json`:
- Pure helper `ensure_statusline(settings: dict) -> (dict, action)` where
  action in {`added`, `exists_other`, `exists_mnemo`}:
  - no `statusLine` key -> add `{type:"command", command:"mnemo statusline",
    padding:0}`; action `added`.
  - `statusLine` present but not mnemo's -> DO NOT clobber; action
    `exists_other` (installer prints guidance to point it at
    `mnemo statusline` manually).
  - already mnemo's -> no-op; action `exists_mnemo`.
- `--no-statusline` / `-NoStatusline` flag skips entirely.
- This is the only write to user settings; it is non-clobbering + idempotent.

**doctor:** add an advisory (`required=False`) check
`check_statusline(settings_json)` -> `[ok]` configured / `[?]` not
configured (with the one-line fix).

**Tests:** `format_statusline` cases (healthy/offline/with-inject);
`ensure_statusline` three branches + idempotency (run twice -> stable);
installer asserts the `statusLine`/`ensure_statusline` path is present;
doctor check unit test.

### Component 3 -- MCP-only-host discovery (Cursor/Continue/Windsurf/...)

**New tool `mnemo_help`** (risk=safe) in `daemon/mnemo/agent_tools.py`
`TOOLS` + handler. Returns a concise capabilities object: what mnemo is,
the key tools (query/search_by_type/traverse/get_node/list_skills...), the
20 workflow skills (or a pointer to `mnemo_list_skills`), and the nudge
"prefer mnemo_query over grep for how/where/why questions." No args.
Surface 29 -> 30.

**Description nudge.** Strengthen the `mnemo_query` (and
`mnemo_search_by_type`) `description` to lead with the grep-nudge so a model
scanning tool descriptions prefers it. Pure string change.

**First-call notice (lightweight).** Module-level "shown" flag in the MCP
path; on the first `mnemo_query` call per server process, attach an additive
`notice` field to the response dict ("mnemo active: N memories; call
mnemo_help for the full surface"). Additive only -- never breaks existing
result parsing. Reset per process (stdio = per host session).

**Tests:** `mnemo_help` in `TOOLS` + `EXPECTED_TOOLS` (30); handler returns
the expected keys + the grep-nudge string; `mnemo_query`/`search_by_type`
descriptions contain the nudge; first-call `notice` present once then
absent. Regenerate `test_mcp_wire_schema_snapshot.py` snapshot.

## 4. Anti-goals

- No new required runtime deps.
- Don't break the working MCP surface (additive only; snapshot regen is the
  guard) or the daemon.
- `auto_inject` stays token-budgeted (<=800); the per-prompt path is
  unchanged except the optional inject-count file write.
- Presence is never spammy: ONE session banner line (opt-out), ONE
  statusline line. No per-prompt user notices. No telephone-home/analytics.
- The statusline must never hang the bar (hard 250ms probe timeout) and
  never open the store in its hot path.

## 5. Definition of Done

**Live operator-green in THIS Claude Code editor:**
1. After a session restart, the session-start banner appears as a
   user-visible notice ("mnemo: N memories ... /mnemo-query to recall").
2. After the installer wires `statusLine` into settings.json + a restart,
   the status bar shows `mnemo {count}` (and `up{N}` after a prompt injects).
3. `mnemo_help` is callable via the MCP surface and returns the capabilities
   object incl. the grep-nudge; `mnemo doctor` shows the new statusline
   check.

**Asserted by construction (NOT live-verifiable in Claude Code):** the
Cursor/Continue/etc. discovery UX -- covered by unit tests on `mnemo_help`,
the description nudges, and the first-call notice. The design doc states
this boundary plainly (lesson #135: operator-green > test-green, and be
honest about which bar each item meets).

**Green gates:** full `uv run pytest` + `ruff check`/`ruff format` clean;
wire snapshot regenerated; surface count 30 everywhere.
