## Summary

**v2.1.3 — hotfix.** The /code page project-card progress bars
were invisible (zero-width strips) after v2.1.1's palette refactor.
v2.1.2 fixed the COLOR but missed the underlying box-model bug.

### Root cause

The /code template marks up each bar as
`<span class="bar-track"><span class="bar-fill"></span></span>`.
`<span>` defaults to `display: inline`, and inline elements
**ignore `width` and `height` per CSS spec**. So the inline
`style="width: 8.7%"` and the CSS `height: 100%` were both
silently dropped -- the fill rendered at 0×0 (DOM-present,
correctly colored, but invisible).

The outer `.bar-track` happened to be sized because it's a grid
item (grid items are blockified by spec). The inner `.bar-fill`
is nested one level deeper, NOT a direct grid item, so it
remained inline.

### Fix

`display: block` on the scoped `.code-project-card-bars .bar-fill`
rule. Also added defensively to the unscoped `.bar-fill` rule so
the dashboard paints regardless of whether the template uses
`<div>` or `<span>`.
