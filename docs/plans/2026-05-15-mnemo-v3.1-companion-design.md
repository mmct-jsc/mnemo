# mnemo v3.1 design — Mnem becomes a real companion

**Codename:** Companion (for real this time).

**Status:** design — brainstormed + the 4 forks decided 2026-05-15.
Implement NEXT session (per user direction). Do NOT implement from this

## 0. Why this exists

v3.0 shipped the spec mechanically but Mnem is not yet a *companion*:
the dock just deep-links to `/chat`, streaming is a raw chunk dump
(not the design §6C `mnemoStreamText` word-smoothing), there's no Chat
nav tab, no compaction, no lazy history, no token visibility, no
bookmarks, `mnemo_run_skill` was never built, and the favicon/logo are
a generic "M". This doc closes every gap the user named and the
design-doc items v3.0 skipped.

## 1. Gap analysis (grounded in the v3.0 code, 2026-05-15)

| Area | v3.0 state | Gap |
|---|---|---|
| Dock | 360px panel with an "Open chat" link | design §6B = working companion panel; user wants full mini-chat, draggable, hidden on /chat |
| Chat nav | none (nav = Nebula/Code/Settings) | no Chat tab |
| Streaming | raw `EventSource` -> `innerHTML` per delta | design §6C mandates `window.mnemoStreamText` (word unit) + a real "working" animation; "smooth like Claude" |
| Thread layout | scrolls, not a fixed pinned viewport | fixed-height, pinned to latest, scroll-up for history |
| History | `GET /v1/chat/<id>` returns ALL messages | lazy paginate (context too high); virtualize |
| Compaction | none | design §8 deferred; user wants it (hybrid) |
| Tokens | providers don't surface `usage` | display running + per-turn + budget bar |
| Bookmarks | none | server-persisted, jump-to |
| Skills | `mnemo_run_skill` NOT implemented (design §3 lists it); 18 skills in `skills/` unused by the agent | add run/list-skill tools |
| Branding | favicon = data-URI "M"; nav brand = text | use the Mnem mark for favicon + logo |

## 2. Decisions locked (the 4 forks)

1. **Dock = full mini-chat.** The dock panel IS a working chat
   (compose + stream + tool chips + inline permission card), a
   per-page conversation that auto-attaches `page_context`. `/chat`
   stays the full-width surface. **Dock is hidden on `/chat`**
   (redundant). One chat implementation, two surfaces (DRY — see §4.6).
2. **Compaction = hybrid.** Anthropic → server-side
   `compact-2026-01-12` beta (preserve compaction blocks verbatim on
   every turn). OpenAI/Google/Ollama → provider-agnostic
   summarize-oldest-turns fallback when estimated input tokens cross a
   configurable threshold; the summary becomes a synthetic pinned
   `system`-role message.
3. **Paginate UI + compaction feeds model.** New
   `GET /v1/chat/<id>/messages?before=<seq>&limit=N`. The UI lazy-loads
   older turns on scroll-up (virtualized, pinned to latest). The
   model's context is bounded by compaction (§2.2) — UI pagination and
   model-context are **separate concerns**.
4. **Server-persisted, rich** bookmarks + tokens. New `chat_bookmarks`
   table; bookmark any message + a jump strip. Running conversation
   token total + per-turn tokens + a budget bar (from the per-provider
   settings budget).

## 3. Architecture deltas

### 3.1 Schema (additive — `SCHEMA_SQL` new table + `_ensure_columns`)

```sql
CREATE TABLE IF NOT EXISTS chat_bookmarks (
  id              TEXT PRIMARY KEY,
  conversation_id TEXT NOT NULL REFERENCES chat_conversations(id) ON DELETE CASCADE,
  message_seq     INTEGER NOT NULL,
  label           TEXT,
  created_at      INTEGER NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_chat_bm_conv ON chat_bookmarks(conversation_id, message_seq);
```
`_ensure_columns` adds to **chat_messages**: `token_in INTEGER`,
`token_out INTEGER`, `cache_read INTEGER` (per-turn usage; NULL on old
rows). To **chat_conversations**: `summary_json TEXT` (compaction
state / running summary), `tokens_total INTEGER NOT NULL DEFAULT 0`.
Store: `add_bookmark`/`list_bookmarks`/`delete_bookmark`;
`list_messages(conv_id, *, before_seq=None, limit=None)` (newest-first
window when bounded); `count_messages`; `bump_tokens`.

### 3.2 Provider protocol delta

Add a 4th event: `('usage', {input_tokens, output_tokens,
cache_read_input_tokens})`, yielded once before `('stop', …)`.
- Anthropic: from `get_final_message().usage` (already cache-aware).
- OpenAI: pass `stream_options={"include_usage": True}`; last chunk
  carries `.usage`.
- Google: `chunk.usage_metadata` (prompt/candidates token counts).
- Ollama: final line `prompt_eval_count` / `eval_count`.
`BaseProvider.stream` contract + every impl + their mocked tests get
the new event. `AgentLoop` records it on the assistant message
(`token_in/out/cache_read`) and `store.bump_tokens(conv)`.

### 3.3 Compaction (`mnemo/compaction.py` + chat loop hook)

- `should_compact(messages, model) -> bool`: cheap token estimate vs
  `config.compaction_trigger_tokens` (default 120_000; settings knob).
- Anthropic path: `AnthropicProvider` gains a `compaction=True` mode →
  uses `client.beta.messages.create(betas=["compact-2026-01-12"],
  context_management={"edits":[{"type":"compact_20260112"}]})` and the
  loop appends FULL `response.content` (compaction blocks preserved —
  the claude-api skill's critical rule).
- Fallback path: `summarize_old_turns(provider, model, msgs)` →
  one terse provider call ("summarize the following conversation
  prefix in <=400 tokens, preserve decisions/ids/citations") → replace
  the summarized prefix with a single pinned `{"role":"system",
  "content":"[earlier conversation summary] …"}` message; persist into
  `chat_conversations.summary_json`.
- The loop calls compaction BEFORE each provider iteration when
  `should_compact`.

### 3.4 REST deltas (`server.py` + `api_schemas.py`)

- `GET /v1/chat/<id>` → returns only the **latest window**
  (`limit=DEFAULT_PAGE`, default 30) + `has_more` + `total`.
- `GET /v1/chat/<id>/messages?before=<seq>&limit=<n>` → older window
  (paginated, design follows `reference_mnemo_pagination.md`: SQL
  LIMIT/OFFSET-by-seq + scalar count; never load-all).
- `GET/POST/DELETE /v1/chat/<id>/bookmarks`.
- `ConversationDetailOut` gains `has_more`, `total`, `tokens_total`;
  `ChatMessageOut` gains `token_in/out/cache_read`, `bookmarked`.

### 3.5 Tools (`agent_tools.py`)

- `mnemo_list_skills()` — **safe**: lists `skills/*/SKILL.md`
  (name + description from frontmatter).
- `mnemo_run_skill(skill_name)` — **confirm**: returns
  `{"_skill": {"name", "guidance": <SKILL.md body>}}`. The loop, on
  this sentinel, injects the guidance as a synthetic
  `system`-role message ("Active skill: <name>\n<body>") for the rest
  of the run (skills are markdown workflow guides, NOT executable fns —
  "running" = the agent reads + follows it, same model the IDE uses).
  18 skills already exist in `skills/` (incl. the v3 `mnemo-doc`).
- This is the design §3 `mnemo_run_skill` finally built + a `list`
  companion. (Deeper "agentic skill templates run end-to-end" =
  design §8 — deferred to v3.2; v3.1 = list + load-into-context.)

### 3.6 UI — one chat component, two surfaces (DRY)

Factor the working `chatPage()` Alpine logic into a single reusable
factory `mnemoChat(opts)` in a shared static JS module
(`static/chat.js`), parameterised by `{surface: 'page'|'dock',
pageContext}`. `/chat` (`chat.html`) renders it full-width;
`base.html`'s dock renders the SAME component in a 380px panel. No
duplicate chat logic.

**Streaming smoothness (design §6C + "like Claude"):** the SSE
`text_delta` handler feeds `window.mnemoStreamText(liveEl, source,
{unit:'word', perUnitDelayMs:~18})` instead of `innerHTML=`. While
`thinking` (no text yet) show a real **working animation** — an
orbiting-dots / shimmer line with a short min-display so it doesn't
flash. Reveal words with the slight cadence Claude uses. (Read
`mnemoStreamText`'s `opts` at `app.js:270` — `unit`, `perUnitDelayMs`,
`formatLine`, reduced-motion aware.)

**Fixed pinned thread:** the thread is a fixed-height fle​x viewport;
new content auto-pins to bottom (only if the user is already near
bottom — Claude behaviour); scrolling up loads history via an
IntersectionObserver sentinel → `?before=<oldest seq>`; cap rendered
turns (virtualization) so a 500-message conversation stays light.

**Bookmarks:** a star affordance per turn → `POST .../bookmarks`; a
slim bookmarks strip (left gutter / minimap) to jump; server-persisted
so it survives reload + device.

**Tokens:** header chip = running `tokens_total` vs the per-provider
budget (a thin budget bar that fills + warns near the cap); per-turn
token count on assistant turns (dim, on hover).

**Draggable dock:** the dock is `position:fixed` but
pointer-draggable; clamp to viewport; **snap to nearest edge** on
release; persist `{x,y,edge}` in `localStorage['mnem.pos']`. Hidden
when `path.startsWith('/chat')`. Click → 380px mini-chat panel
(`mnemoChat({surface:'dock'})`), re-click/Esc collapses;
`mnem.docked` (closed|open|pinned) still respected.

### 3.7 Mnem branding

- New `static/mnem/mark.svg` — a SIMPLIFIED Mnem glyph (the teal disc +
  3-node constellation, no eyes/mouth) that reads at 16px.
- `base.html`: replace the data-URI favicon with
  `/static/mnem/mark.svg` (+ a 32px PNG fallback `mark-32.png` if
  needed for older browsers). Nav brand becomes the mark + "mnemo"
  wordmark. `/chat` welcome + dock already use the full mascot.
