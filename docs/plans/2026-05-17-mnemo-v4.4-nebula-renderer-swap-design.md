# mnemo v4.4 — Nebula Renderer Swap (sigma.js v3) — design

**Status:** design (validated forks confirmed with the user via structured question, 2026-05-17)
**Decisions locked:** sigma.js v3 + graphology; "best + most beautiful + most optimized, follow our design theme" (C1 tokens); split from v4.3.2 (its own chapter).
**Hard constraint (gotcha 31 / `reference_cosmos_gl_nebula` TOMBSTONE):** do NOT re-attempt cosmos.gl config/wiring/perpetual-motion. v4.4 *replaces* cosmos.gl outright — that is the only sanctioned path past the documented ceiling.

## Why

The companion can compute "what's related" (the off-renderer tools
`mnemo_session_nodes` / `mnemo_highlight_nodes` survive in v3.2) but
**cannot show it on the graph**: `@cosmos.gl/graph@3.0.0-beta.8`
(`graph.html:16-24`) has a documented closed ceiling — it cannot do
stable + smooth-camera + alive + node-highlight simultaneously;
`setConfig` after `new Graph` is ignored, `setPointPositions` is
ignored while the sim is inactive, the converge-and-stop halts the
rAF loop, and every attempt to wire a highlight listener re-triggered
the v2.6.8 freeze (reverted; gotcha 31). v3.2/C3 made the *wording*
honest ("surfaced in the side panel, not the graph"). v4.4 makes the
graph itself deliver: **real node highlight + full interaction**, by
swapping the renderer.

## Engine: sigma.js v3 + graphology

Why sigma.js v3 (over custom regl/PixiJS, react-force-graph/three.js,
G6):

- **Highlight is first-class.** `setSettings({ nodeReducer,
  edgeReducer })` recomputes per-node/per-edge display every frame —
  greying the rest while spotlighting a set is a pure data change, no
  renderer-internal-state trap (the exact thing cosmos couldn't do).
- **Full interaction is built-in.** `clickNode` / `enterNode` /
  `downNode` events, `camera.animate()` (zoom/pan/center), node drag,
  hover — no bespoke hit-testing.
- **Scale.** WebGL; proven at 10k+ nodes (we have ~11.8k). No
  converge-and-stop pathology — layout runs to a settled state then
  the renderer just paints (matches the v2.6.6 "pin-all-static"
  outcome we already shipped, but with interaction on top).
- **No Node build.** sigma + graphology ship UMD/ESM bundles loadable
  exactly like cosmos is today (`<script type="module">` from a
  pinned CDN or vendored static) — preserves the "no Node toolchain"
  stack rule.
- **graphology** is the in-memory graph model + layout
  (`graphology-layout-forceatlas2` worker) — compute positions once
  (or reuse the server layout cache, see Data flow), then render
  static + interactive.

YAGNI: no 3D, no clustering UI, no physics playground — replace the
renderer, keep the existing 3-panel shell, scope logic, and data
contract.

## Architecture (slots into the existing shell — minimal blast radius)

Unchanged (reused verbatim):
- `{% block layout %}` + `<main class="full">` + `.nebula-shell`
  3-panel grid (C1 `calc(100vh - var(--topbar-h))`), `x-data="nebula()"`.
- `/ui/graph-data` endpoint + server layout cache
  (`_graph_scope_key` + `_graph_fingerprint`, `routes.py:58-74`) +
  workspace scoping (`active_ws.project_keys`). The data contract
  (nodes, edges, positions, scope) does **not** change.
- DOM label overlay learning is *obsoleted* — sigma renders labels
  natively (themed); the overlay is removed (net simplification).

Replaced (the renderer layer only):
- `graph.html:16-24` cosmos ESM import → sigma.js v3 + graphology
  (pinned ESM, or vendored to `mnemo/ui/static/vendor/` to remove the
  CDN runtime dependency — decide in plan phase 1; vendoring is the
  "most optimized/robust" choice and matches the no-build rule).
- `<div id="cy-nebula">` stays as the sigma container (id reused so
  the guard test's `cy-nebula` token survives).
- The cosmos-specific traps (`setConfig`/`setPointPositions`/
  converge-and-stop comments, `setLinkColors` paint hack, the
  never-cool sim) are deleted; sigma's reducer/event model replaces
  them.

## Capabilities unlocked

1. **Real node highlight.** A `highlight(nodeIds)` Alpine method sets
   a reactive `Set`; the `nodeReducer` renders in-set nodes at full
   `--accent` glow and the rest dimmed (alpha ~0.15) — never hidden
   (same "grey, don't hide" intent as the old `_ui` highlight). The
   companion's `mnemo_highlight_nodes` (`agent_tools.py:854`) `_ui`
   sentinel now drives THIS — closing the gotcha-31 / C3-honesty loop;
   the C3 "side panel, not the graph" wording can be reverted to the
   truthful "highlighted in the graph" once this lands (a v4.4 task).
2. **Full interaction.** click node → select + cite side-panel +
   `@mnemo-cite`; hover → neighbour-emphasis + label; drag → reposition
   (ephemeral, not persisted — YAGNI); camera zoom/pan/`center on
   node`; click-empty → deselect (mirrors the cytoscape capability-test
   lesson `feedback_cytoscape_minified_core`).
3. **Search-to-highlight.** the existing rail/search selecting a node
   → `camera.animate` to it + highlight (the interaction the user
   couldn't get).

## Design theme (C1 — non-negotiable polish bar)

Every sigma visual is driven from the C1 single-source tokens +
`palette.py`, never hardcoded:
- node color = `palette.TYPE_COLORS[type]` (the existing single-source
  per-type palette, already a Jinja/JS/CSS contract — sigma reads
  `window.MNEMO_TYPE_COLORS`).
- accent/glow = `--accent` / `--accent-soft`; background = `--bg`
  starfield (keep the loved gradient/starfield shell);
  dim/edge = `--border-soft`/`--fg-dim`; labels = `--fg` / `--mono`.
- selection ring, hover halo, highlight glow tuned for the dark theme
  (the "beautiful" bar). No new palette — extend `palette.py` if a
  graph-specific role is needed (one row, the proven model).

## Error handling / fallbacks

- sigma init failure or empty in-scope set → the existing empty/error
  panel (reuse `nebula()`'s current states).
- Large scope: keep the server layout cache (positions precomputed,
  fingerprinted); if absent, graphology forceatlas2 in a worker with a
  bounded iteration cap, then freeze (no perpetual sim — the cosmos
  lesson).
- Vendored bundle missing → graceful "renderer asset not found"
  message (CI guards the vendored file exists).
