## Summary

One PR ships two tagged releases:

- **v2.0.0 -- "Code Intelligence"** -- every registered `code_repo`
  source becomes a typed graph (modules / functions / classes /
  methods / routes / components / endpoints) with Tier 1 universal
  ingestion, Tier 2 Python call-graph resolution, and Tier 3
  framework extractors for FastAPI / Flask / Express / React.
  Adds a `/code` UI family and 7 new skills.
- **v2.1.0 -- "Nebula" UI overhaul** -- promotes `/graph` to a
  three-panel resizable shell (VSCode-style file tree | force-
  directed canvas | Prism-highlighted detail panel) with a
  vibrant palette, glow halos, silky transitions, and an
  expandable filter bar. `/code` landing redesigned with type-
  breakdown bars and method-colored endpoint cards.

18 commits, all on `release/2.0.0`. Tags `v2.0.0` and `v2.1.0`
already point at their release commits; the late-session polish
(commits `04afd8b` -> `c326434`) sits between `v2.1.0` and HEAD.

## Headline capabilities

### v2.0 -- Code Intelligence

- **Typed code graph.** `code_module` + `code_function` +
  `code_class` + `code_method` + `code_route` + `code_component` +
  `code_endpoint` node types; `imports` / `defines` / `method_of`
  / `calls` / `routes_to` / `at_endpoint` edges with per-edge
  confidence (0.0..1.0).
- **Tier 1 ingestion** via tree-sitter (8 bundled grammars +
  10 lazy). Python gets full structural extraction; other
  languages get a module-only fallback.
- **Tier 2 Python call-graph resolver.** Stack-Graphs-inspired
  scope resolution with constructor + `self`/`this` lookups.
  Same-file edges 0.95 confidence; cross-file via imports 0.8.
- **Tier 3 framework extractors.** FastAPI + Flask + Express
  (backend) + React (frontend). Cross-stack join via shared
  `code_endpoint` URI anchors.
- **Source auto-router** with dry-run preview + 50,000-file
  safety ceiling (the structural fix for "Duyen-class"
  registration mistakes).
- **`/code` UI family.** Landing, project overview with
  file-grouped declarations, function detail with 2-hop
  ego-network, cross-stack sitemap.
- **7 new code skills.** `mnemo:explore-codebase`,
  `mnemo:trace-call`, `mnemo:trace-route`,
  `mnemo:explain-design`, `mnemo:debug-with-code`,
  `mnemo:why-is-this-here`, `mnemo:impact-analysis`.

### v2.1 -- Nebula UI

- **Three-panel resizable shell** at `/graph` (tree | canvas |
  detail). Drag gutters to resize; widths persist to
  localStorage. Top-bar nav renamed `Graph` -> `Nebula`.
- **Vibrant palette.** Saturated neon-on-velvet, glow halos on
  every node (per-type color via Cytoscape underlay), starfield
  + bloom backdrop, marching-ants animation on highlighted
  edges, pulse on the selected node.
- **VSCode-style file tree** with inline SVG icons (folder
  chevron + colored corner-fold per file extension), active-file
  accent stripe.
- **Prism.js syntax highlighting** in the detail panel with
  per-token Nebula palette overrides.
- **Silky transitions** -- cubic-bezier 200-300ms easing on
  chips / gutters / tree / panels / buttons; layout switches
  cross-fade (140ms out -> snap -> 220ms in + 320ms camera fit)
  without per-node tweening.
- **Smart filter bar.** Text search, per-type chip toggles,
  confidence slider with tick marks, hop selector, layout
  buttons (force / rings / tree / grid). Force layout snapshots
  its initial positions so round-tripping through other layouts
  always returns to the same shape.
- **Three deselect paths** -- canvas tap, Escape key, "x"
  close button in detail panel.
- **`/code` landing redesign** -- full-page width, hero title
  with gradient text, project cards with type-breakdown bars,
  method-colored endpoint grid (GET=green, POST=blue, PUT=amber,
  PATCH=violet, DELETE=crimson), expandable truncation
  ("show N more" buttons), inline search filter that bypasses
  the cap.

## Performance

- `animate: false` on all relayouts inside `cy.batch()` -- one
  render pass per layout switch instead of 400 per-node tweens.
- `textureOnViewport` + `hideEdgesOnViewport` +
  `hideLabelsOnViewport` for cheap pan/zoom.
- Click-detail batched to 95-100ms steady (was 300+ms).
- Pulse moved off `setInterval` to Cytoscape's RAF scheduler;
  only the selected node animates, not its edges.
